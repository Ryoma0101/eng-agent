# プロジェクト・コンテキスト

## 1. プロジェクト概要

**プロジェクト名**：EnQuest

**チーム**：WILLとTNDのハッカソンチーム3

**期間**：2026年ハッカソン（1週間集中開発）

---

## 2. コンセプト

> **「英作文を、孤独な学習からエキサイティングな競技（スポーツ）へ」**

AIによる**動的な問題生成**と**精密な自動採点**を組み合わせ、ユーザー同士がビジネス英語の「論理構成力」と「表現力」をスコアで競い合うプラットフォーム。

### ビジョン

2031年、生成AIと協働して英語で仕事をすることが当たり前になる。
しかし既存の英語学習は「問題を解く」だけで、実務に直結していない。

このプラットフォームでは、**スコアだけでなく、英語で仕事を遂行する力を鍛える**。

---

## 3. 市場の課題（ブルーオーシャン）

### 既存英語学習の限界

| 課題                         | 詳細                                                                       |
| ---------------------------- | -------------------------------------------------------------------------- |
| **採点の浅さ**               | 単純な正誤判定のみで、ビジネスに不可欠な「説得力」や「論理性」の評価が弱い |
| **モチベーション維持の困難** | 一人で添削を受けるだけでは継続が難しい                                     |
| **コミュニティの欠如**       | Kaggleのような「競い合い、高め合う」環境がない                             |
| **実務との乖離**             | 学習内容がビジネスシーンで直接活かされない                                 |

### 市場機会

- ビジネス英語学習者：数百万人規模
- グローバル競争の激化による英語需要の増加
- AI採点技術の成熟により「スコア可視化」が実現可能に

## 4. 主要機能案

### ① AI動的クエスト生成

**概要**  
OpenAI APIを活用し、最新のビジネスニュースや商学的なケーススタディを基にした英作文問題を毎日自動生成。

**問題文の特徴**

- 時事ネタ系・ビジネスケース
- 論理構成を要求する形式
- 指定字数範囲（例：150～200 words）

**例**

```
「円安が輸出企業に与える影響について、3つの観点から論じなさい」
「新興国市場での競争戦略について説明してください」
```

### ② 多角的なAI自動採点（Scoring Algorithm）

**採点方式**  
単なるスペルチェックではなく、以下の多次元スコアを算出。

#### スコア計算式

$$\text{Score} = w_1 \cdot G + w_2 \cdot L + w_3 \cdot C + w_4 \cdot F$$

#### 採点項目

| 項目            | 内容                           | 配点目安 | 重み $w_i$ |
| --------------- | ------------------------------ | -------- | ---------- |
| **G (Grammar)** | 文法の正確さ、文構造           | 0-25点   | 0.25       |
| **L (Logic)**   | 論理展開の整合性、説得力       | 0-25点   | 0.30       |
| **C (Context)** | 文脈に適した語彙、ビジネス用語 | 0-25点   | 0.25       |
| **F (Fluency)** | 自然な言い回し、読みやすさ     | 0-25点   | 0.20       |

**最終スコア**  
0～100点の標準化スコア

**実装方針**

- **LLM採点**：GPT-4で上記4項目を採点
- **ルール補正**：字数不足、指定フォーマット未達で減点

### ③ リアルタイム・グローバルランキング

**ランキング表示**

- **デイリーランキング**：当日（JST）の全ユーザーのスコアトップN
- **ウィークリーランキング**：週間（月～日 JST）の総合スコア
- **総合ランキング**：累積スコア

**ユーザー称号システム**  
Kaggle風メダル制度

| 称号        | 条件                         |
| ----------- | ---------------------------- |
| Grandmaster | デイリーランキング1位3回以上 |
| Master      | 平均スコア95点以上           |
| Expert      | 平均スコア85点以上           |
| Candidate   | 50提出以上                   |

## 5. ユーザーストーリー

### ペルソナ1：ビジネス英語学習者（25～35歳、社会人）

**背景**  
グローバル企業でキャリアを積みたいが、英作文スキルに自信がない。

**ニーズ**

- 実務的な文章力を手短に習得したい
- 他者と競争してモチベーションを保ちたい
- AIによる正確な評価が欲しい

**期待される行動**

1. 毎日のクエストに挑戦（1クエストあたり3～5分）
2. ランキングを確認してレベルアップを目指す
3. "Grandmaster"称号を目指してコンテスト参加

### ペルソナ2：学生（18～22歳）

**背景**  
大学で英語を学んでいるが、従来の学習が退屈。

**ニーズ**

- ゲーム感覚で楽しく学びたい
- 友達と競争したい
- リアルタイムフィードバックが欲しい

**期待される行動**

1. 友達をシェアして招待
2. 日々のランキング変動を楽しむ
3. SNS投稿で成績を自慢

## 6. 提供価値（Value Proposition）

### ユーザーへの価値

| 価値               | 詳細                                           |
| ------------------ | ---------------------------------------------- |
| **スコア化**       | 実務英語の「強み・弱み」が数値で可視化される   |
| **競争**           | ランキング＆称号で継続モチベーションが湧く     |
| **実務直結**       | ビジネスケース中心の問題で実践的な力が身につく |

- **ネットワーク効果**：ユーザー数増加でランキング・コミュニティ価値が増大
- **データ資産**：採点データから学習者の特性を分析可能
- **B2B展開**：企業研修プラットフォームへの展開可能性

## 7. スコープ（ハッカソン1週間版）

### 実装する機能

- ✅ ユーザー認証（Firebase Auth）
- ✅ クエスト表示（hardcoded問題）
- ✅ AI採点実行（Route Handler）
- ✅ スコア保存（Firestore）
- ✅ デイリーランキング表示
- ✅ 基本的なUI（shadcn/ui）

### スコープ外（今後の拡張）

- ❌ ウィークリー集計の自動化
- ❌ SNS共有機能
- ❌ 称号システムのUI
- ❌ 外部ニュースAPI連携
- ❌ ユーザースタディングの分析ダッシュボード
